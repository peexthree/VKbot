# 🏗 Архитектура проекта

В этом документе описана техническая структура **VK Esoteric Bot** и логика взаимодействия его компонентов.

---

## 🛰 Высокоуровневая схема

```text
[ Пользователь VK ]
       |
       v
[ VK API (Callback / LongPoll) ]
       |
       v
[ VKbottle (Dispatcher) ] <------> [ Upstash Redis (FSM + Lock + Cache) ]
       |
       v
[ Business Logic (Modules) ] <---> [ Supabase (User Data + History) ]
       |
       v
[ AI Service (Gemini API) ] <----> [ Personas & Base Prompts ]
       |
       v
[ Output (Message / PDF / VK Pay) ]
```

---

## 🛠 Ключевые компоненты

### 1. Обработка событий (Event Handling)
*   **Main Loop:** В файле `main.py` инициализируется `vkbottle`, который слушает события. Мы используем Webhooks для продакшена (Render) и Long Polling как запасной вариант.
*   **Middlewares:** `ThrottleMiddleware` защищает систему от флуда, ограничивая частоту запросов от одного пользователя.

### 2. Управление состоянием (FSM)
*   Мы отказались от стандартного in-memory FSM в пользу **Redis**. Это позволяет боту «помнить», на каком шаге регистрации или гадания находится пользователь, даже после перезагрузки сервера.
*   Функции: `get_fsm_step`, `set_user_state`.

### 3. Интеграция с ИИ (Gemini)
*   Бот использует **Gemini 2.0 Flash/Pro**.
*   **Ротация ключей:** Ключи берутся из списка `GEMINI_API_KEYS`, что позволяет обходить лимиты бесплатных тарифов.
*   **Vision:** Для хиромантии бот передает изображения в ИИ через `inline_data` (base64).

### 4. Генерация PDF
*   Используется связка **Jinja2** (шаблоны) + **WeasyPrint** (рендер в PDF).
*   Для ускорения работы используется `pdf_semaphore`, ограничивающий количество одновременных генераций для экономии RAM.

### 5. База данных (Supabase)
*   Хранит профили пользователей, балансы, историю разборов и коллекцию карт (Гримуар).
*   Взаимодействие идет через REST API Supabase (библиотека `aiohttp`).

---

## 🔄 Жизненный цикл платной услуги

1.  **Запрос:** Пользователь нажимает «Купить» в каталоге.
2.  **Проверка:** Модуль `payments` проверяет баланс в Supabase.
3.  **Оплата:**
    *   Если энергии достаточно — баланс списывается сразу.
    *   Если нет — пользователю предлагается ссылка на VK Pay.
4.  **Генерация:** Запускается `execute_generation`.
5.  **Доставка:** Результат отправляется в VK, а копия сохраняется в историю.
6.  **Fail-safe:** Если ИИ вернул ошибку, энергия автоматически возвращается на баланс пользователя.

## 🛡 Безопасность и Лимиты
*   **VK API Flood Control:** Реализована задержка в `ghost_edit`.
*   **Redis Locks:** Предотвращают race conditions при одновременных нажатиях кнопок.
*   **Sentry:** Сбор всех необработанных исключений в реальном времени.
