# 🤝 Руководство по разработке (CONTRIBUTING)

Мы рады, что вы хотите внести свой вклад в развитие **VK Esoteric Bot**! Этот проект объединяет магию эзотерики и мощь ИИ. Чтобы процесс был эффективным, следуйте этим правилам.

---

## 🛠 Подготовка окружения

1.  **Форкните** репозиторий.
2.  Установите зависимости (рекомендуется Python 3.11+):
    ```bash
    pip install -r requirements.txt.old
    ```
3.  Настройте `.env` на основе `.env.example`.
4.  Используйте **Ruff** для форматирования кода:
    ```bash
    ruff format .
    ```

---

## 📜 Правила написания кода

*   **Асинхронность:** Все функции ввода-вывода (IO) должны быть `async`.
*   **Типизация:** Используйте аннотации типов. Проверяйте код через `mypy`.
*   **Логирование:** Используйте `loguru` для записи важных событий и ошибок.
*   **Стейты:** При работе с FSM (состояниями) всегда используйте Redis-хранилище через функции в `database/states.py`.

---

## 🚀 Процесс внесения изменений

1.  **Ветка:** Создайте ветку с префиксом `feature/` или `fix/`.
2.  **Тесты:** Если вы добавляете новую логику, напишите для неё тест в папке `tests/`.
    ```bash
    PYTHONPATH=. python -m pytest tests/
    ```
3.  **Документация:** Обновите README.md, если изменились переменные окружения или структура проекта.
4.  **Pull Request:** Опишите, что именно вы изменили и почему. Приложите скриншоты, если затронут UI.

---

## 🎭 Работа с промптами ИИ

Промпты — это «сердце» наших персонажей.
*   При изменении системных инструкций в `prompts/base.py` убедитесь, что вы не удалили обязательные технические требования (например, запрет на форматирование звездочками).
*   Тестируйте новые промпты на разных персонажах (`skin`), чтобы сохранить их уникальный стиль.

---

## ❗ Что НЕЛЬЗЯ делать

*   **Хардкодить секреты:** Никогда не добавляйте токены или ключи в код. Только через `.env`.
*   **Спамить сообщениями:** Старайтесь редактировать старые сообщения бота вместо отправки новых.
*   **Игнорировать Flood Control:** Всегда учитывайте лимиты VK API.
